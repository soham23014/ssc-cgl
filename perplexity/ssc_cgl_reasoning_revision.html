High importance

Analogy

Question shape: A : B :: C : ?
Goal: keep the same relation from the first pair into the second pair.
How to solve:
  • Read both words or numbers carefully.
  • Identify the relation type: meaning, function, part-whole, category, number, sequence, opposite, or worker-tool relation.
  • Test the same relation on the third term and every option.
  • Keep the one option that preserves the exact relation, not just a loose similarity.
Fast shortcut: Say the relation aloud in one phrase, such as “tool used by person”, “young one of animal”, “opposite meaning”, or “same arithmetic change”.
Relation patterns you must know:
  • Meaning relation: doctor : hospital.
  • Function relation: pen : write.
  • Part-whole relation: wheel : car.
  • Category relation: rose : flower.
  • Young one relation: cat : kitten.
  • Opposite relation: north : south.
  • Number relation: 2 : 4, 3 : 9.
  • Alphabet relation: A : C, B : D.
PYQ-style practice forms:
  • Person : place :: person : place.
  • Animal : young one :: animal : young one.
  • Object : use :: object : use.
  • Number : number :: number : number.
  • Opposite word pairs.
  • Letter positions and matching shifts.
Common mistakes:
  • Choosing an answer because it is related, but not in the same way.
  • Forgetting the direction of relation.
  • Not checking whether the pair is word-based or number-based.
  • Jumping to options before naming the relation.

Trick: always identify the relation type first, then search for the same relation in the answer choice.

High importance

What SSC usually asks in Analogy

Question families:
  • Word relation analogy.
  • Number analogy.
  • Letter analogy.
  • Object-to-function analogy.
  • Animal-young-one analogy.
  • Profession-place analogy.
How to answer each family:
  • Word relation: find the exact semantic link.
  • Number relation: check square, double, addition, subtraction, product.
  • Letter relation: convert letters to positions and compare the shift.
  • Function relation: ask “what does it do?”
  • Young one relation: find the child form of the animal.
Training examples with logic:
  • Doctor : Hospital :: Teacher : School — profession to workplace.
  • Cat : Kitten :: Dog : Puppy — adult to young one.
  • 2 : 4 :: 3 : 9 — square relation.
  • North : South :: East : West — opposite direction.
  • Pen : Write :: Knife : Cut — tool and function.

Best habit: write the relation in your head as a short label before checking answers.

Classification

Question shape: choose the item that does not belong.
How to solve:
  • First check whether the items belong to one group.
  • Then compare one property only: category, use, shape, value, position, or sequence.
  • Look for the single item that breaks the group rule.
Fast shortcut: Do not force many logic patterns. Ask one question: “What is common in the 3 correct items?”
Common PYQ forms:
  • One odd word out.
  • One odd number out.
  • One odd figure out.
  • One odd alphabet out.
  • One odd place or object out.
Example logic:
  • Apple, Mango, Banana, Carrot → carrot is a vegetable, others are fruits.
  • 2, 3, 5, 9 → 9 is not prime.
  • Square, Circle, Triangle, Table → table is not a shape.
  • Delhi, Mumbai, India, Kolkata → India is a country, others are cities.
Common mistakes:
  • Trying to find multiple answers.
  • Using a hidden logic that is not actually needed.
  • Ignoring a simple property like prime, odd/even, or category.
  • Thinking too long instead of checking the basic mismatch.
High importance

Classification: quick method

Step method:
  1. Group the items by form.
  2. Group the items by value or use.
  3. Group the items by sequence or order.
  4. If one item fails all simple groups, it is the answer.
Data checks often used in SSC:
  • Prime numbers vs composite numbers.
  • Even vs odd numbers.
  • Squares and cubes.
  • Animals, fruits, countries, tools, shapes.
  • Alphabet order and reverse order.
Timed practice idea:
  • Do 10 odd-one-out questions in one sitting.
  • Stop after 15 seconds per question.
  • Use the simplest grouping rule first.

Series

Question shape: find next number, next letter, or missing term.
How to solve:
  • Check the difference between consecutive terms.
  • If that fails, check second difference.
  • Then check multiplication, division, or alternating terms.
  • For letters, convert to positions and repeat the same test.
Fast shortcut: The easiest pattern is usually the right one. Do not jump to complex patterns before checking arithmetic.
Common patterns in SSC:
  • Arithmetic progression.
  • Increasing or decreasing difference.
  • Multiplication chain.
  • Square/cube based pattern.
  • Alternate chain pattern.
Practice logic:
  • 2, 5, 8, 11, ? → add 3 each time.
  • 3, 6, 12, 24, ? → multiply by 2.
  • 1, 4, 9, 16, ? → squares.
  • A, C, F, J, ? → letter shifts are increasing.
Common mistakes:
  • Skipping the simplest arithmetic pattern.
  • Forgetting to check every alternate term.
  • Mixing series and coding logic too early.
  • Not writing the pattern clearly.
High importance

Series: pattern family

Pattern family checklist:
  • Difference pattern.
  • Second difference pattern.
  • Multiply or divide pattern.
  • Square/cube pattern.
  • Odd-even alternate pattern.
How to read quickly:
  • Write 3 gaps, not only the numbers.
  • Look at change in gaps.
  • If one sequence is too regular, use the regular rule.
Short practice set:
  • 7, 10, 13, 16, ?
  • 5, 10, 20, 40, ?
  • 1, 3, 6, 10, ?
  • 2, 6, 18, 54, ?
High importance

Alphabet Series

Question shape: missing letter, next letter, or shifted letter.
How to solve:
  • Convert letters into numbers.
  • Find the shift size.
  • Check if the shift is constant or increasing.
  • Also check reverse alphabet if needed.
Fast shortcut: A=1, B=2, ..., Z=26 must become automatic. Reverse alphabet is also important.
Common forms:
  • Forward alphabetical sequence.
  • Backward alphabetical sequence.
  • Fixed shift sequence.
  • Increasing shift sequence.
Practice logic:
  • A, D, G, J, ? → shift by 3.
  • Z, X, V, T, ? → shift backward by 2.
  • B, E, H, K, ? → shift by 3.
Common mistakes:
  • Forgetting reverse positions.
  • Counting letters incorrectly.
  • Mixing forward and backward shifts.
  • Not checking whether the shift changes.
High importance

Alphabet: deeper use

Advanced checks:
  • Use position sum or difference.
  • Use reverse alphabet positions.
  • Use pairwise letter movement.
Examples of hidden logic:
  • Same gap between letters.
  • Two gaps increasing step by step.
  • Letters mirrored from the end of alphabet.
Practice tip:
  • Say the position of each letter fast.
  • Turn the positions into simple math.
High importance

Missing Number Series

Question shape: find the missing term in a number chain.
How to solve:
  • Check if the chain is arithmetic.
  • Check if the chain is multiplicative.
  • Check if the terms are square or cube based.
  • Check if two chains are alternating.
Fast shortcut: Do not stay stuck on one term. First test the common families: AP, GP, squares, cubes, alternating.
Question families:
  • Simple missing term.
  • Square or cube missing term.
  • Alternate chain missing term.
  • Increasing gap pattern.
Practice logic:
  • 2, 6, 12, ?, 30
  • 1, 8, 27, ?, 125
  • 4, 9, 16, ?, 36
  • 10, 20, 15, 30, 25, ?
Common mistakes:
  • Trying many patterns at once.
  • Not testing square/cube logic.
  • Ignoring alternating chains.
  • Wasting time on one stubborn question.
High importance

Missing Number: quick attack

Attack order:
  1. Difference.
  2. Ratio/multiplication.
  3. Square relation.
  4. Cube relation.
  5. Alternate chain.
Why this works:
  • SSC often uses simple families.
  • The answer is usually hidden in the gap pattern.
  • Fast filtering saves time in the exam.
High importance

Coding-Decoding

Question shape: find the code or decode the word.
How to solve:
  • Check if each letter is shifted by a fixed number.
  • Check whether the word is reversed.
  • Check whether letters are mapped directly.
  • Check whether the whole word pattern repeats.
Fast shortcut: test shift, reverse, direct mapping, and word pattern in that order.
Common forms:
  • Letter shift code.
  • Reverse word code.
  • Pattern substitution code.
  • Position-based code.
Practice examples:
  • CAT → DBU type shift code.
  • WORD → DROW type reverse code.
  • Same-step letter movement.
  • Repeated word structure mapping.
Common mistakes:
  • Mixing all possible methods at once.
  • Missing the direction of shift.
  • Not checking reverse order.
  • Forgetting exact letter position change.
High importance

Coding-Decoding: deeper set

Extra checks:
  • Alphabet gap check.
  • Reverse order check.
  • Odd-even position rule.
  • Same code for repeated similar words.
Practice route:
  • First decode one word.
  • Then test the same rule on the second word.
  • If both fit, the rule is correct.
Medium importance

Blood Relation

Question shape: find family relation from statements.
How to solve:
  • Draw a family tree.
  • Mark each person with generation level.
  • Mark gender when the statement says son, daughter, brother, sister, father, mother, uncle, aunt, husband, or wife.
  • Read the final relation from the diagram.
Fast shortcut: If you do not draw it, you can easily mix up parent, sibling, and cousin relations.
Common forms:
  • Direct family relation.
  • Two-step relation.
  • Three-step chain relation.
  • Male-female mixed relation.
Practice examples:
  • A is the son of B, B is the father of C. Find relation of A to C.
  • Who is the mother’s brother?
  • Who is the sister’s son?
  • Find relation after two links in the chain.
Common mistakes:
  • Solving without a diagram.
  • Missing generation direction.
  • Confusing brother with cousin.
  • Forgetting gender clues.
Medium importance

Blood Relation: drawing method

Drawing steps:
  1. Write the starting person.
  2. Move up for parents.
  3. Move sideways for siblings.
  4. Move down for children.
  5. Use arrows and labels for clarity.
Why it matters:
  • Long relation questions become simple.
  • Visual map removes confusion.
Medium importance

Direction

Question shape: find final direction or final position.
How to solve:
  • Fix the starting point.
  • Use arrows for each move.
  • Turn exactly as the question says.
  • Calculate final distance or facing direction from the diagram.
Fast shortcut: draw every step. Do not imagine the path loosely.
Common forms:
  • Turn-based movement.
  • Final facing direction.
  • Shortest distance from start.
  • Complicated path with multiple turns.
Practice examples:
  • Walk 5 m north, turn right, walk 3 m.
  • Turn left twice and find final facing.
  • Find shortest distance from origin after a path.
Common mistakes:
  • Confusing left and right.
  • Forgetting the starting direction.
  • Skipping the turn in the diagram.
  • Mixing facing direction with final position.
Medium importance

Direction: quick map

Quick map rule:
  • Use north, south, east, west clearly.
  • Mark left and right according to current facing, not page side.
Best practice style:
  • Always draw small compass marks.
  • Write each move step by step.
Medium importance

Order Ranking

Question shape: find rank from left, rank from right, or total number of people.
How to solve:
  • Check whether the rank is from left or right.
  • If both ranks are given, use the total formula.
  • Remember total = left rank + right rank - 1.
  • Check whether the question asks position or actual count.
Fast shortcut: never mix position with final rank. First identify the side.
Common forms:
  • Rank from left.
  • Rank from right.
  • Total number of students.
  • Rank after one person changes position.
Practice examples:
  • A is 7th from left and 10th from right. Find total.
  • B is 5th from left in a row. Find right rank if total is known.
  • Find position after adding one person to the row.
Common mistakes:
  • Mixing left and right ranks.
  • Forgetting -1 in total formula.
  • Thinking position and rank are same.
  • Not reading the side carefully.
Medium importance

Ranking: quick formula use

Formula rule:
  • Total people = left rank + right rank - 1.
  • If total is given and one side is known, the other side can be found directly.
Practice style:
  • Write both ranks clearly before solving.
  • Check whether the row is fixed or changed by conditions.

  • Alphabet Series: positions must be automatic.
