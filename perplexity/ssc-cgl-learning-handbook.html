Subject 1: Reasoning

Reasoning is not “IQ.” It is pattern filtering under time pressure. The winning habit is to identify the question family within 3–5 seconds and apply the standard solution route without emotional guessing.

Tier 1 demands speed and fast elimination. Tier 2 usually pushes more layered conditions, tighter distractors, and slightly longer logical chains.
AnalogyRelationship matching between words, numbers, letters, or figures.
▶

Basic explanation

An analogy asks: what exact relation connects the first pair, and which option repeats the same relation? If you don’t define the relation in one line, you are just guessing.

Identification method

  • Check type first: meaning, classification, cause-effect, part-whole, tool-work, degree, number operation, letter gap.
  • Never look at options before stating the relation in words.
  • If 2+ relations seem possible, choose the most precise one.

SSC patterns

  • Word analogy: country-capital, animal-young one, object-material, profession-tool.
  • Number analogy: square/cube, prime jumps, digit sum, multiplication plus/minus constant.
  • Alphabet analogy: positional difference, reverse alphabet, pair symmetry.

Shortcut and topper trick

In number analogy, first test the quickest families: square/cube, prime, digit sum, product, ratio, difference. In alphabet analogy, write positions only when mental mapping fails.

Step-by-step solving

  • Convert the first pair into a rule.
  • Classify rule as semantic, arithmetic, or positional.
  • Apply same rule to third term.
  • Reject options that fit partially but not exactly.

Common mistakes

  • Choosing broad relation instead of exact relation.
  • Ignoring order; A:B may not equal B:A.
  • Using forced logic after seeing options.

Tier 1 vs Tier 2

Tier 1 keeps the relation more visible. Tier 2 may mix double relations, e.g. category plus degree, or number operation plus constant. PYQ trend: analogy remains a stable scoring area because pattern families repeat heavily.

Revision note: define the relation in six words or fewer.Rule > option temptation.
Classification / Odd One OutThree follow one rule; one breaks it.
▶

Core idea

Odd one out is reverse analogy. Find the common property of most options and spot the intruder.

Fast approach

  • Look for category first, then sub-category, then property.
  • In numbers, test parity, prime/composite, perfect square, divisibility, digit sum.
  • In letters, check same gap pattern.

SSC repeated types

  • Animal-bird-fish mixes.
  • State-capital or ministry-person mismatch.
  • One number breaking common arithmetic property.

Mistakes

Students often pick a weird-looking option instead of proving the common rule among the other three.

Tier angle

Tier 1 uses direct families. Tier 2 may require a second-level filter, like all are metals but one is alloy, or all are squares of primes except one.

Number SeriesMissing term or wrong term through numerical pattern.
▶

Zero to basic

A series is solved by checking the simplest consistent pattern first. Most losses happen because students jump to fancy logic before testing basic operations.

Order of checking

  • Difference
  • Second difference
  • Ratio/division
  • Alternate terms
  • Square/cube/primes/Fibonacci-style additions
  • Digit operations

SSC patterns

  • Alternating + and - with growing constants.
  • Multiply then add/subtract.
  • Prime number movement.
  • Square-based terms: 4, 9, 16 with offsets.

Topper trick

Write the gap row once. If gaps don’t talk, split into odd-even positions immediately. That single habit saves stupid time.

Example

3, 8, 15, 24, ? → gaps are 5, 7, 9, so next gap 11; answer 35.

Common trap

Forcing one rule on the entire series when the actual pattern is alternate terms.

Tier 1 vs Tier 2

Tier 1 is usually one-step recognizable. Tier 2 can chain two operations or use wrong-number detection requiring back-checking on both sides.

Alphabet SeriesLetter positions, gaps, reverse order, skipping patterns.
▶

Must memorize

Know A=1 to Z=26 cold. Also know reverse positions: A=26, B=25... That is non-negotiable.

Solving route

  • Convert letters to positions only when needed.
  • Check constant gap, alternating gap, reverse movement, pair symmetry.
  • For mixed letters and numbers, separate both tracks.

Important rules

  • Opposite letters sum to 27: A+Z, B+Y, C+X.
  • Middle letters: M=13, N=14.
  • Vowel positions worth memorizing: A 1, E 5, I 9, O 15, U 21.

Mistakes

Wrong numbering, ignoring reverse sequence, and missing alternate-term patterns.

Coding-DecodingOld pattern coding, matrix coding, letter/word transformation.
▶

Basic explanation

Coding means original data is transformed through a rule. Your job is to detect the rule family before testing the option.

Main types

  • Letter shift coding.
  • Word substitution coding.
  • Number coding by letter positions.
  • New pattern coding with symbols.
  • Matrix or language coding from given statements.

Identification trick

  • Compare first and last letters.
  • Check length change.
  • See whether coding is direct, reversed, alternate, or grouped.
  • For statement-based coding, mark common word/common code pairs first.

SSC topper trick

In language coding, start from the pair of statements that shares exactly one common word. That isolates one code fast.

Common mistakes

  • Using one decoded relation globally without verifying all positions.
  • Ignoring repeated letters and order.
  • Confusing direct code with word-position code.

Tier angle

Tier 1 asks direct coding. Tier 2 prefers statement-based or multi-step transforms. PYQ trend: language coding and matrix coding reward calm deduction, not speed panic.

Blood RelationFamily tree logic, generation, gender, and relation tracing.
▶

Basic

Blood relation is just structured family mapping. Confusion starts only when you try to do it in your head without a rough tree.

Step method

  • Take central person as reference.
  • Mark generation levels vertically.
  • Use symbols: M/F, up for parents, down for children, same line for siblings/spouse.
  • Resolve pronouns carefully.

Repeated SSC patterns

  • Pointing/family introduction questions.
  • Coded relation symbols.
  • Mixed statements with ‘only son’, ‘maternal’, ‘paternal’.

Shortcut

Ignore names. Track only relation labels and gender certainty. Names are noise.

Mistakes

  • Assuming gender where not given.
  • Confusing maternal and paternal sides.
  • Forgetting reference person changes in the last statement.

Tier difference

Tier 1 is shorter and mostly direct. Tier 2 may combine coded relations or layered family chains. Previous questions repeat the same family-tree skeletons with different wording.

Direction & DistanceMovement, turning, coordinates, final direction and shortest distance.
▶

Core idea

Treat movement as coordinate changes. North-south is vertical, east-west is horizontal. Stop imagining roads dramatically; draw a clean path.

Standard solving approach

  • Assume start at origin.
  • Turn right/left based on current facing, not fixed north.
  • Compute final coordinates.
  • Shortest distance = straight-line distance; for perpendicular legs use Pythagoras.
Distance = √(x² + y²)

SSC patterns

  • Simple turns and facing.
  • Distance after multiple moves.
  • How far and in which direction from start.
  • Shadow-based direction with time of day.

Common mistakes

Turning according to map north instead of current facing direction, and mixing total path length with shortest distance.

Order & RankingPosition from top/bottom, left/right, middle and total count.
▶

Basic formula logic

If one rank from top and one from bottom are known for the same person, total people = top rank + bottom rank - 1.

Total = Rank from top + Rank from bottom - 1

How to solve

  • Write positions linearly.
  • Watch whether the row/queue direction is left-right or front-back.
  • Middle position exists only when total count is odd.

SSC traps

  • Interchanging left and right after turning.
  • Using formula for two different people.
  • Missing that the row has been rearranged.

Tier approach

Tier 1 direct formula. Tier 2 may combine ranking with arrangement or coded order.

Sitting ArrangementLinear, circular, facing in/out, variable constraints.
▶

Truth

This chapter destroys time because students don’t structure information. One clean diagram beats rereading clues five times.

Solving order

  • Choose framework: line or circle.
  • Place absolute positions first.
  • Then fixed relations: immediate left/right, second to left, opposite.
  • Then negative clues: not next to, not at end, not facing.
  • Test remaining options systematically.

Facing rule

  • Facing center: left is clockwise, right is anticlockwise.
  • Facing outside: left is anticlockwise, right is clockwise.

Shortcut

For circular arrangement, fix one person at top to remove rotational duplicates unless orientation is tied to a real direction.

Mistakes

  • Changing facing rule midway.
  • Not marking uncertain placements separately.
  • Trying to mentally solve without a structured grid.

Tier difference

Tier 1 uses fewer clues. Tier 2 can combine professions/colors/people. Previous SSC arrangements are usually moderate, not CAT-level monsters, so overcomplicating is stupidity.

SyllogismAll/Some/No relations between sets.
▶

Basic explanation

Syllogism is set logic. Ignore English sentence feel; translate statements into set relations.

Core rules

  • All A are B → A lies fully inside B.
  • No A is B → no overlap.
  • Some A are B → at least one overlap.
  • Some A are not B → at least one outside.

SSC solving method

  • Draw quick Venn if direct.
  • For conclusion checking, accept only guaranteed truth.
  • Possibility-based conclusion requires checking whether any case allows it without violating statements.

Common mistakes

  • Treating “some” as “some but not all.”
  • Concluding converse: All A are B does not mean all B are A.
  • Using real-world knowledge over statement logic.

Tier angle

Tier 2 may include possibility cases or more premises. PYQ trend is stable: if your basic set logic is clean, this remains a gift chapter.

Statement & ConclusionDoes the conclusion logically follow from given statements?
▶

Simple explanation

A conclusion must be logically forced by the statement. If it is merely likely, emotionally appealing, or generally true, reject it.

Approach

  • Read only statement facts.
  • Ignore external knowledge.
  • Check whether the conclusion is definite, probable, or unrelated.
  • Extreme words like all, only, always need strict support.

Pattern recognition

SSC often traps students with socially sensible conclusions that are not logically guaranteed. Logic does not care about your feelings.

Mistakes

Importing assumptions from common sense and confusing cause with correlation.

Statement & AssumptionHidden premise on which the statement depends.
▶

Concept

An assumption is an unstated idea taken for granted. If that idea is false, the statement weakens or collapses.

Check method

  • Use the “negation test” mentally.
  • If negating the assumption makes the statement pointless or weak, the assumption likely holds.
  • Advertisements, advice, and policy statements often hide assumptions.

SSC repeated forms

  • Advice statements.
  • Government or company announcements.
  • Appeals and recommendations.

Trap

Do not confuse supporting fact with necessary assumption. An assumption is required, not merely helpful.

Venn DiagramSet overlap in word groups and data relation logic.
▶

Basic

Venn Diagram questions test category overlap, containment, and separation. Convert words into sets, not stories.

Method

  • Identify if groups are disjoint, overlapping, or subset.
  • Use minimum-circle structure first.
  • For numerical data, fill the central intersection first when all totals are given.

Repeated SSC patterns

  • Profession-gender-activity groups.
  • Sports/club/language overlaps.
  • Logical relation among three words.

Shortcut

When exact relation is unknown between two groups but both are inside a bigger set, use the safer diagram that preserves both possibilities if options ask only broad relation.

CalendarOdd days, leap year, day-date relation.
▶

Core rules

  • Ordinary year = 365 days = 1 odd day.
  • Leap year = 366 days = 2 odd days.
  • Leap year if divisible by 4, but century years must be divisible by 400.

How to solve

  • Count odd days from a reference point.
  • Know month odd days by practice, not by panic.
  • For same calendar year, compare leap status and start day.

Memory trick

Month days: 30 for Apr, Jun, Sep, Nov; Feb 28 or 29; rest 31. Stop relying on finger tricks if they slow you down.

Mistakes

Ignoring leap day impact for January/February vs later months and mishandling century years.

ClockAngle, coincidence, opposite positions, time relation.
▶

Need-to-know formulas

Angle = |30H - 5.5M| Coincidence roughly every 65 5/11 minutes Opposite hands when angle = 180°

Usage

  • Hour hand moves continuously, not jump-wise.
  • Minute hand speed = 6° per minute.
  • Hour hand speed = 0.5° per minute.

Tier approach

Tier 1 usually direct angle finding. Tier 2 may ask mirror clock or relative position logic with more conditions.

Common mistakes

Using 30H only and forgetting the hour hand moves with minutes too.

Mathematical OperationsSymbol replacement and operator interchanges.
▶

Basic

Operators are replaced by symbols. Your job is to remap symbols correctly before calculating.

Solving route

  • Write symbol meaning first.
  • Replace entire expression.
  • Apply BODMAS normally.
  • For options, test quickly by simplification.

Mistake

Doing the calculation before remapping completely.

SSC trick

In symbol-interchange questions, if one interchange fixes all operator order positions, verify with one side fast and skip full expansion.

Missing NumberMatrix and diagram number logic.
▶

Pattern families

  • Row/column sum or product.
  • Diagonal relation.
  • Outer numbers combine to inner number.
  • Square or difference relation.

Identification

Check basic operations in order: +, -, ×, ÷, square, digit sum. Most SSC missing numbers use two-step arithmetic, not magic.

Example

If top row is 2, 3, 11 and pattern is 2×3+5 = 11, verify same logic in the next row before choosing.

Mistake

Not cross-checking the guessed rule across all available rows/boxes.

Counting FiguresCount triangles, squares, rectangles, lines in complex diagrams.
▶

Core idea

Count systematically by size and orientation. Random visual counting guarantees missed figures.

Method

  • Count smallest figures first.
  • Then combine adjacent figures to make larger ones.
  • Track orientation separately when diagonals exist.

Topper trick

Touch-count with a finger or pointer path on screen. Your eyes lie when figures overlap.

Mistakes

Double counting same figure or skipping rotated/combined shapes.

Mirror Image & Water ImageLateral inversion vs top-bottom inversion.
▶

Rule

Mirror image reverses left and right. Water image reverses top and bottom. Stop mixing them; that error is embarrassingly common.

Approach

  • Look for asymmetrical markers first: arrows, slants, letter openings.
  • Use edge-point comparison rather than full figure imagination.
  • Remember letters/digits with symmetry: A, H, I, M, O, T, U, V, W, X, Y may reduce effort.

Mistake

Rotating mentally instead of reflecting.

Paper Folding, Cutting & Embedded FiguresHidden pattern tracking in folded or complex visual forms.
▶

Paper folding

Each fold mirrors cuts onto the folded layers. Unfold in reverse order mentally.

Embedded figure method

  • Search the target figure by line-angle pattern, not full shape.
  • Ignore extra lines; only required structure matters.
  • Focus on unique corners, intersections, and lengths.

SSC pattern

Questions usually use one or two folds, not extreme visual complexity. Calm scanning beats speed panic.

Mistake

Expecting exact visual cleanliness in embedded figures; the target may be disguised inside clutter.

Dice / Cubes & CuboidsOpposite faces, painted surfaces, visible faces, counting blocks.
▶

Dice basics

  • Faces seen together are adjacent.
  • Faces never seen together in valid views may be opposite.
  • Use common-face method for two views.

Cubes and cuboids basics

For painted cube questions, classify mini-cubes by exposed faces: corner cubes = 3, edge cubes = 2, face-center cubes = 1, inner cubes = 0.

If cube is divided into n parts per edge: Corner = 8 Edge = 12(n-2) Face center = 6(n-2)² Inner = (n-2)³

Usage

Use only when all outer surfaces are painted. If some faces are unpainted, adjust face counts before applying formulas blindly.

Mistakes

Applying full-paint formulas to partial-paint problems and confusing opposite with adjacent faces.

Figure Series & Non-verbal ReasoningShape progression, rotation, counting, shading, symmetry, rule sequence.
▶

Question types

  • Rotation and reflection.
  • Addition/deletion of elements.
  • Movement of dots/lines.
  • Shading alternation.
  • Embedded sequence in complex forms.

Pattern recognition framework

  • Observe count change.
  • Observe direction change.
  • Observe fill/unfill pattern.
  • Observe alternate frame logic.

Fast trick

For non-verbal series, look at what changes, not what stays same. The static part is noise.

Tier difference

Tier 1 mostly direct visuals; Tier 2 may add denser distractors. Still, the logic families repeat more than students admit.

Subject 2: Quantitative Aptitude

SSC Quant is won by ruthless formula recall plus decision speed. You do not need Olympiad-level genius. You need to know which formula to apply, when to avoid long calculation, and how not to commit basic arithmetic suicide.

Tier 1 rewards speed, option logic, simplification, and percentage-based shortcuts. Tier 2 expects stronger algebraic handling, cleaner setup, and fewer calculation errors.
SimplificationBODMAS, fraction-decimal-percent conversion, fast arithmetic reduction.
▶

Concept

This chapter is basically controlled arithmetic. If your number sense is weak, you bleed marks everywhere else too.

Fast calculation methods

  • Cancel before multiplying.
  • Convert mixed fractions to improper form early.
  • Use common denominators only when needed.
  • Memorize square values up to 30 and cube values up to 20.

Important values

1/2=50%, 1/3=33.33%, 1/4=25%, 1/5=20%, 1/6=16.67%, 1/8=12.5%, 1/9=11.11%, 1/11=9.09%

SSC trick

Approximation is useful only if options are wide enough. If options are tight, do exact work; fake speed becomes negative speed.

Mistakes

Breaking BODMAS order, sign errors, and failing to reduce fractions before multiplication.

Number SystemDivisibility, factors, multiples, remainders, HCF, LCM, base properties.
▶

Core concepts

  • Natural, whole, integers, rational and irrational numbers.
  • Prime/composite, even/odd.
  • Factor count and remainder logic.

Formulas

If N = p^a q^b r^c, Number of factors = (a+1)(b+1)(c+1) Sum of factors = (p^(a+1)-1)/(p-1) × ... HCF × LCM = Product of two numbers

Divisibility rules to memorize

  • 3 or 9 by digit sum.
  • 4 by last two digits.
  • 8 by last three digits.
  • 11 by alternating sum difference.

When to use what

Use prime factorization for HCF/LCM and factor count. Use Euclid-style logic or direct division when numbers are ugly but small.

Common mistakes

Mixing HCF and LCM interpretation in word problems and mishandling negative remainders.

Tier approach

Tier 1: divisibility, factor count, unit digit, quick LCM/HCF. Tier 2: deeper remainder, cyclicity, and structured number property questions.

PercentageBase chapter for profit-loss, SI-CI, DI, ratio conversion.
▶

Concept

Percentage is normalized comparison per hundred. If you are slow here, half of arithmetic becomes slow.

x% of y = (x/100) × y

Fast methods

  • 10% shift decimal one place.
  • 5% = half of 10%.
  • 1% = divide by 100.
  • Successive percentage change uses net formula.
Net change = a + b + ab/100

Important values

  • 12.5% = 1/8
  • 16.67% = 1/6
  • 20% = 1/5
  • 25% = 1/4
  • 33.33% = 1/3
  • 37.5% = 3/8
  • 66.67% = 2/3

Mistakes

Applying percentage increase and decrease symmetrically. A 20% increase followed by 20% decrease does not return to original.

Ratio & ProportionComparison, scaling, direct and inverse relation.
▶

Basic

Ratio compares quantities of same unit. Proportion means two ratios are equal.

a:b = c:d ⇒ ad = bc

Usage

  • Use ratio when total or parts are scalable.
  • Use proportion when cross-product relation is clean.
  • Add/subtract in ratio only after converting to common scale.

SSC tricks

  • Compound ratio in partnership and work questions.
  • Convert ratio to fraction for percentage links.
  • Use common multiplier for age and mixture problems.

Mistakes

Adding terms of ratio directly to actual quantities without defining the common multiplier.

Average & AgeMean concept, age-shift logic, replacement and group averages.
▶

Average formula

Average = Sum of observations / Number of observations

Average is balance point, not mystical arithmetic.

Age solving approach

  • Age difference stays constant.
  • Use present age as variable when relation is time-shifted.
  • For average change after inclusion/replacement, use total sum change.

SSC variations

  • Average of consecutive numbers.
  • Change in average after new member joins.
  • Father-son and sibling age ratio with time shift.

Mistakes

Using ratio as absolute age and forgetting the same number of years applies to all people.

Profit & Loss / DiscountCP, SP, MP, marked price discounts and gain-loss relation.
▶

Formulas

Profit = SP - CP Loss = CP - SP Profit% = Profit/CP × 100 Loss% = Loss/CP × 100 Discount = MP - SP Discount% = Discount/MP × 100

When to use which base

Profit and loss percentages use CP as base. Discount uses MP as base. Mixing bases is a rookie mistake.

SSC special tricks

  • Use ratio method: 20% profit means CP:SP = 100:120 = 5:6.
  • Successive discounts: net discount = a + b - ab/100.
  • Dishonest dealer questions often combine weight and price.

Mistakes

Taking profit % on SP and discount % on CP. That is flatly wrong.

Tier difference

Tier 1 direct formulas and percentage links. Tier 2 may combine marked price, discount, GST-like adjustments, or false weights.

Simple Interest & Compound InterestInterest growth, annual/half-yearly compounding, CI-SI comparison.
▶

Formulas

SI = PRT / 100 Amount(SI) = P(1 + RT/100) Amount(CI) = P(1 + R/100)^T CI = Amount - P

Usage rules

  • Use SI when interest applies only on principal.
  • Use CI when interest is added back each period.
  • For half-yearly compounding: rate half, time double.

SSC special points

  • Difference between CI and SI for 2 years often tested.
  • For 2 years, difference = P(R/100)^2.
  • For 3 years, use standard expanded difference carefully.

Mistakes

Applying yearly formula directly to half-yearly compounding and forgetting amount vs interest distinction.

Time & Work / Pipe & CisternWork efficiency, combined rates, leak and fill logic.
▶

Core logic

Work is rate-based. Stop using random LCM tricks unless you understand why.

If A finishes in x days, one day work = 1/x Combined work = sum of rates

LCM method

Take total work as LCM of individual days when it simplifies arithmetic. Then efficiency = work/day.

Pipe usage

  • Filling pipe adds rate.
  • Leak/outlet subtracts rate.
  • Alternate opening questions require segment-wise work addition.

Topper trick

If men, women, boys are related by efficiency ratio, convert everything into one unit before touching days.

Mistakes

Adding time directly instead of rates, and mishandling “works together for some days then one leaves” type questions.

Time Speed Distance / Train / Boat & StreamUniform motion, relative speed, length crossing, upstream-downstream.
▶

Main formula

Speed = Distance / Time Distance = Speed × Time Time = Distance / Speed

Train formulas

  • Crossing pole: time = train length / speed.
  • Crossing platform: time = (train + platform length) / speed.
  • Two trains opposite: relative speed adds.
  • Same direction: relative speed subtracts.

Boat & stream

Downstream speed = b + s Upstream speed = b - s Still water speed = (D + U)/2 Stream speed = (D - U)/2

Speed conversion

1 m/s = 18/5 km/h 1 km/h = 5/18 m/s

Mistakes

Using total speed instead of relative speed, forgetting unit conversion, and ignoring total distance covered by both objects.

Mixture & Allegation / PartnershipWeighted average of values and profit sharing by investment-time.
▶

Allegation rule

Cheaper quantity : Dearer quantity = (Dearer price - Mean price) : (Mean price - Cheaper price)

When to use

  • Use allegation when two components form a mean value.
  • Use weighted average when there are more than two components or added/removed quantity over stages.

Partnership rule

Profit share ∝ Capital × Time

If capital changes mid-year, split time blocks. Don’t fake it.

Mistakes

Using capital ratio alone when investment durations differ.

AlgebraIdentities, linear and quadratic equations, simplification of expressions.
▶

Must-know identities

(a+b)^2 = a^2 + 2ab + b^2 (a-b)^2 = a^2 - 2ab + b^2 a^2-b^2 = (a-b)(a+b) (a+b)^3 = a^3+b^3+3ab(a+b) a^3-b^3 = (a-b)(a^2+ab+b^2)

Usage

  • Expand only when simplification is required.
  • Factorize when cancellation is possible.
  • In quadratic equations, first bring to standard form.

SSC repeated forms

  • Identity-based numerical substitution.
  • Roots of quadratic.
  • Equation in one variable.

Mistakes

Sign errors in expansion and unnecessary expansion when factorization is faster.

Coordinate Geometry & Data InterpretationDistance, section basics, graphs, tables, pie and bar data.
▶

Coordinate formulas

Distance between (x1,y1) and (x2,y2) = √[(x2-x1)^2 + (y2-y1)^2] Midpoint = ((x1+x2)/2, (y1+y2)/2)

DI exam method

  • Read units first.
  • Mark what is asked: ratio, difference, percentage, average.
  • Do not read all data blindly; target the relevant cells/bars only.
  • Use approximation only if options permit.

Tier 1 vs Tier 2

Tier 1 DI is simpler and often percentage-based. Tier 2 can push more table extraction and two-step calculations.

Mistakes

Ignoring units, mixing percentage points with percentage change, and over-reading irrelevant data.

Subject 3: English

SSC English is a rule-based scoring section disguised as language skill. Stop treating it like vague “fluency.” The exam repeats grammar patterns, common error zones, and predictable vocabulary families.

Daily 30–40 minute model: 10 min grammar rules, 10 min error spotting + sentence improvement logic, 10 min vocabulary memory, 5–10 min reading or para-jumble practice.
Parts of SpeechNoun, pronoun, verb, adjective, adverb, preposition, conjunction, interjection, article use.
▶

Simple explanation

Parts of speech tell what job a word is doing in a sentence. Same word can play different roles depending on use.

Exam importance

Error spotting and sentence improvement become easier when you first identify the function of the underlined word or phrase.

Memory technique

  • Noun names.
  • Pronoun replaces noun.
  • Verb shows action/state.
  • Adjective qualifies noun.
  • Adverb qualifies verb/adjective/adverb.

Common mistake

Calling every descriptive word an adjective and forgetting adverbs can modify adjectives too.

Noun & Pronoun RulesNumber, gender, case, agreement, pronoun reference.
▶

Noun rules

  • Some nouns look plural but take singular verb: news, politics, mathematics.
  • Some have same singular/plural: deer, sheep, aircraft.
  • Collective noun may be singular as one unit.

Pronoun rules

  • Pronoun must agree with noun in number and person.
  • Each/every/everyone/anyone takes singular pronoun in traditional SSC patterns.
  • Pronoun reference must be clear.

Repeated SSC errors

  • One should do his duty / one should do one’s duty type construction.
  • Each of the boys has done his work.
  • Neither of the two has completed his task.

Common mistakes

Pronoun without clear antecedent and mismatch between singular subject and plural pronoun.

Voice & NarrationActive-passive transformation and direct-indirect speech.
▶

Voice basics

Passive voice shifts object to subject position. Only transitive verbs form passive normally.

Active: Subject + Verb + Object Passive: Object + be + V3 + by + Subject

Narration basics

  • Change reporting verb as per sentence type.
  • Backshift tense when reporting verb is past, unless universal truth/current validity holds.
  • Change pronoun and time/place words by context.

SSC repeated rules

  • Imperative: ordered/requested/advised + to-infinitive.
  • Question: asked/inquired + if/whether or question word.
  • Exclamation: exclaimed with joy/sorrow.

Mistakes

Forcing passive for intransitive verbs and changing tense mechanically without context.

Articles, Preposition, Conjunction, Adjective, AdverbHigh-frequency grammar zones.
▶

Articles

  • Use “a” before consonant sound, “an” before vowel sound.
  • Use “the” for specific/unique things, superlatives, rivers, mountain ranges, newspapers.
  • Zero article with many abstract/uncountable nouns in general sense.

Preposition memory

  • Interested in, good at, capable of, prefer to, differ from, comply with.
  • Memorize as phrases, not isolated words.

Adjective/Adverb traps

  • Adjective after linking verbs: feel bad, seem happy.
  • Hard vs hardly, late vs lately, near vs nearly.

Mistakes

Choosing article from spelling instead of sound and translating Hindi structures directly into English prepositions.

Error Spotting & Sentence ImprovementApplied grammar diagnosis.
▶

Solving method

  • Check subject-verb agreement.
  • Check tense and verb form.
  • Check article, preposition, pronoun.
  • Check parallelism and comparison.
  • Check idiomatic usage.

SSC repeated rules

One of the + plural noun + singular verb; no sooner...than; hardly/scarcely...when; prefer X to Y; between two = better, among many = best context matters.

Common mistakes

Changing a grammatically correct but unfamiliar sentence because it sounds odd.

Exam approach

Do not hunt every error at once. Scan in a fixed order. Structured scanning beats language ego.

Fill in the Blanks, Cloze Test, Reading Comprehension, Para JumbleContext-based English topics.
▶

Fill/Cloze method

  • Check grammar fit first.
  • Then meaning and collocation.
  • Read one line before and after the blank.
  • In cloze, keep tone consistency across passage.

Reading comprehension method

  • Read questions first if passage is factual.
  • Mark keywords and line references mentally.
  • Choose answer supported by passage, not opinion.

Para jumble method

  • Find opening sentence: general intro, no pronoun dependency.
  • Find mandatory pairs: pronoun-reference, article-noun, cause-effect, chronology.
  • Find closing sentence: conclusion/result.

Mistakes

Choosing the most beautiful sentence instead of the one linked logically with neighbors.

  • Study words in clusters: praise words, criticism words, government words, personality words.
  • Use root/prefix/suffix: bene, mal, anti, auto, omni, graph, logy.

5 new words, 5 old revisions, 5 idioms twice a week. That is boring but effective. Fancy methods usually fail.
